24/*
25 * cap_load_scattered - load a self-addressed binary card deck into a flat image.
26 *
27 * Each data card (>= 11 columns) is decoded (TC_COLBIN) and read as a
28 * self-describing record: b[8]=LL, b[9..10]=load address (big-endian),
29 * b[11..11+LL]=LL+1 payload bytes. The payload is *scattered* to `addr` in
30 * `image` (a caller-provided 65536-byte buffer). Cards are gated by the deck's
31 * dominant 8-byte prefix (cols 0-7); if no dominant prefix is found, any
32 * self-consistent record is accepted (loose). Bytes not written keep their
33 * existing value in `image` (caller should pre-zero or pre-seed as desired).
34 *
35 * This mirrors the gdis `--image` depunch so a .cap deck loads identically
36 * whether depunched offline or fed to the emulator directly. It honours each
37 * card's embedded load address (the deck format), independent of the (separate,
38 * cycle-faithful) card-reader bootstrap.
39 *
40 * On success returns the number of cards loaded (>0) and sets *lo/*hi to the
41 * lowest/highest address written (so [*lo, *hi] is the populated span; entry is
42 * conventionally *lo). Returns -1 on parse error or if nothing loaded. `mode` is
43 * an `enum transcode_mode` (pass TC_COLBIN for binary decks).
44 */
